# Configuration

## Config Files

| File | Location | Purpose |
|------|----------|---------|
| `blockhost.yaml` | `/etc/blockhost/` | Server keypair, public secret, admin wallet, fund manager settings |
| `web3-defaults.yaml` | `/etc/blockhost/` | Blockchain config (explorer URL, signer URL, network, payment token) |
| `admin-commands.json` | `/etc/blockhost/` | Admin command definitions (port knocking, etc.) |
| `addressbook.json` | `/etc/blockhost/` | Role-to-wallet mapping (admin, server, hot, dev, broker) |
| `revenue-share.json` | `/etc/blockhost/` | Revenue sharing configuration (dev/broker splits) |
| `vms.json` | `/var/lib/blockhost/` | VM database (IPs, NFT state, ownership) |
| `engine.json` | `/usr/share/blockhost/` | Engine manifest (identity, wizard plugin, constraints) |
| `.testing-mode` | `/etc/blockhost/` | Presence = testnet mode (addresses start with `3` instead of `9`) |

## web3-defaults.yaml

Blockchain configuration for the Ergo engine:

```yaml
blockchain:
  explorer_url: https://api-testnet.ergoplatform.com
  signer_url: http://127.0.0.1:9064
  network: testnet
  payment_token: ""
```

| Key | Description |
|-----|-------------|
| `explorer_url` | Ergo explorer API base URL (mainnet: `https://api.ergoplatform.com`) |
| `signer_url` | ergo-relay signing service (default `http://127.0.0.1:9064`) |
| `network` | `mainnet` or `testnet` |
| `payment_token` | 64-char hex token ID for stablecoin payments (empty = native ERG) |
| `submit_url` | Optional override for transaction submission endpoint |

No API key required â€” Ergo explorer is public. Signing is handled by ergo-relay (local Rust binary), not a remote node.

## blockhost.yaml

General server and fund manager configuration:

```yaml
server_public_key: "03..."               # Compressed secp256k1 pubkey (66 hex chars)
public_secret: blockhost-ergo-testnet    # Challenge message prefix
contract_address: ""                      # Subscription guard P2S address (set by deploy-contracts)

admin:
  wallet_address: "3W..."                # Admin wallet (Ergo Base58)
  shared_key: "<64-char hex>"            # HMAC key for admin commands
  credential_nft_id: ""                  # Admin NFT token ID
  max_command_age: 300                   # Seconds; nonces older than this are pruned

fund_manager:
  fund_cycle_interval_hours: 24
  gas_check_interval_minutes: 30
  min_withdrawal_nanoerg: 50000000000    # 50 ERG minimum before collection
  gas_low_threshold_nanoerg: 5000000000  # 5 ERG â€” triggers warning
  gas_swap_amount_nanoerg: 20000000000   # 20 ERG â€” target swap amount (stub)
  hot_wallet_gas_nanoerg: 1000000000     # 1 ERG â€” target hot wallet balance
  server_stablecoin_buffer_nanoerg: 50000000000  # Stablecoin buffer (token base units)
```

All monetary thresholds are in **nanoERG** (1 ERG = 1,000,000,000 nanoERG).

## Addressbook

Role-to-wallet mapping in `/etc/blockhost/addressbook.json`. All addresses are Ergo Base58:

```json
{
  "admin":  { "address": "3W..." },
  "server": { "address": "3W...", "keyfile": "/etc/blockhost/deployer.key" },
  "dev":    { "address": "3W..." },
  "broker": { "address": "3W..." }
}
```

Entries with `keyfile` can sign transactions. The `hot` entry is auto-added on first fund cycle.

## deployer.key

The server wallet's raw private key. Contains a 64-character hex string (32 bytes). Generated by `bhcrypt generate-mnemonic` (which derives the key from a BIP39 mnemonic via BIP32 EIP-3 path `m/44'/429'/0'/0/0`).

The mnemonic is displayed to the operator during wizard setup for backup purposes but is not stored on disk â€” only the derived raw key is written to `deployer.key`.

## Revenue Sharing

Configured in `/etc/blockhost/revenue-share.json`:

```json
{
  "enabled": true,
  "total_bps": 2000,
  "recipients": [
    { "role": "dev",    "bps": 1000 },
    { "role": "broker", "bps": 1000 }
  ]
}
```

`total_bps` is basis points (100 bps = 1%). `recipients[].role` maps to addressbook keys. The recipient `bps` values must sum to `total_bps`. The legacy `total_percent` / `percent` format is also accepted (converted to bps internally).

## admin-commands.json

Defines the commands available for on-chain admin dispatch (HMAC-authenticated R4 register protocol):

```json
{
  "commands": {
    "knock": {
      "action": "knock",
      "params": {
        "allowed_ports": [22],
        "default_duration": 300
      }
    }
  }
}
```

`action` must match a registered action handler in `src/admin/index.ts`. Currently supported: `knock` (opens an iptables rule for a port for a configured duration via the root agent).

## .testing-mode

Presence of `/etc/blockhost/.testing-mode` switches the engine to testnet:
- Addresses use testnet prefix (`3...` instead of `9...`)
- Fund cycle runs every 10 minutes (instead of 24 hours)
- Distribution steps are skipped (preserves deployer ERG for testing)
- Reconciliation runs every 2 minutes (instead of 5)
